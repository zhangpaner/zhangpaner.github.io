JavaScript事件循环？

发表于2022-09-28|更新于2025-03-18|JavaScript基础
|字数总计:279|阅读时长:1分钟|阅读量:

JavaScript 事件循环过程：

  • JavaScript 是单线程的，意味只有一个主线程，在 js 引擎执行过程中，是从上到下执行的。
  • 事件分为同步任务和异步任务，会首先将同步任务放进主线程的执行栈中(call stack)，异步任务会放在宿主环境中(浏览器/Node)，等待合适时机会被放进任务队列当中。每当执行栈中的任务执行完毕后，就会去调用任务队列中的任务放进执行栈执行。重复这个过程，就是事件循环。
  • ES6 新增 Promise 后，异步任务又分为宏任务(浏览器/Node)和微任务(javascript)，首先执行微任务队列再执行宏任务队列。每次执行宏任务之前都会先查找是否有微任务，有则优先执行。(微任务可插队宏任务！)
